Conseils de gestion du portefeuille —

Divergence live vs backtest —
Pour chaque set : ce que fait ton live contre ce que promet son backtest — tous deux à 0,1 % du solde par trade. Le verdict s'appuie sur un test statistique (z sur le taux de réussite, t sur l'espérance) — pas sur un simple écart brut.
En attente de trades live et de backtests importés.

EA silencieux —
Chaque set a une cadence connue par son backtest. Si le silence dure trop longtemps, l'EA est probablement planté, mal configuré ou MT5 déconnecté. La probabilité affichée est celle d'observer un tel silence si le set tradait normalement.
En attente de trades live et de backtests importés.

Coût d'exécution réel —
Un backtest ne connaît ni ton vrai spread, ni le slippage, ni la latence du VPS. On compare ici ton trade moyen réel à celui attendu, et on isole la part explicitement facturée (commissions + swaps).

Projection du compte —
Depuis ton solde actuel, on rejoue des milliers de fois les trades du portefeuille backtesté (à ton risque) pour encadrer les mois à venir. C'est une distribution de possibles, pas une prévision.
Le compte démo est le banc d'essai permanent des sets hors réel : chaque set y est jugé contre son backtest, exactement comme les sets réels dans « Contrôle live ». Les recommandations de rotation sont visibles en continu mais se tranchent à la revue semestrielle — sauf urgence (drawdown > 130 % de la référence).

Rotation réel ↔ démo —
Règles validées : un set réel sort si son drawdown dépasse 110 % de sa référence backtest (130 % = urgence, sans attendre la revue). Un set démo est promouvable s'il est conforme à son backtest (≥ 20 trades) et décorrélé des sets réels (corrélation < 0,3 idéale, < 0,6 acceptable faute de mieux — mesurée sur les P&L quotidiens des backtests). Le plafond de sets en réel dépend du compte actif (27 avec les Swing sniper, 21 pour Only Nomad).
